# scores — プロジェクト固有データの置き場（例）

プロジェクトで扱う **データアセット** を置く場所の例。

「scores」はもともと「楽譜（score）」を想定した名前だが、楽曲データに限らず
**プロジェクト特有のデータ一式**を置く場所として読み替えてよい。用途に合わせて
リネームしても問題ない（例: `assets/`, `data/`, `maps/`, `scenarios/` など）。

## こんなときに使う

- ゲームのマップデータ・シナリオ・パラメータ
- センサのキャリブレーション値
- 楽譜 / MIDI / サウンドパッチ
- テストケース用のサンプルデータ
