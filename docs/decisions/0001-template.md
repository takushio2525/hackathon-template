# ADR-0001: ADR テンプレート

**ステータス**: Template

## 背景

Architecture Decision Record（ADR）の記入テンプレート。重要な設計判断を残すためのフォーマットを揃える。

新しい判断を記録するときは，このファイルをコピーして `NNNN-<short-title>.md` にリネームして書き始めること。

## 決定事項

ここに「何を決めたか」を1〜2文で書く。

## 理由

なぜその決定に至ったか。代替案を比較する。

## 結果・影響

決定によって発生するトレードオフ，影響範囲。後で読み返したときに「なぜこの判断をしたか」が分かるように書く。

## ステータスの種類

- `Proposed`: 議論中
- `Accepted`: 採用決定
- `Rejected`: 却下
- `Deprecated`: 過去の決定（現在は別の方法を採用）
- `Superseded by ADR-NNNN`: 別の ADR で置き換えられた
