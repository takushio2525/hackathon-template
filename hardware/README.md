# hardware — ハードウェア資料の置き場（例）

回路図・配線図・部品表・データシートを置くディレクトリ。ハードを使わない班は
丸ごと削除してよい。

## 構成

| ディレクトリ | 内容 |
|---|---|
| `schematics/` | 回路図（KiCad 想定。`.kicad_sch` / `.kicad_pro` / `.kicad_pcb` 等） |
| `wiring/` | 配線図（Fritzing / 写真 / 画像） |
| `datasheets/` | 使用部品のデータシート PDF |

## 部品表

[parts_list.md](parts_list.md) を参照。購入品と実験室貸出品は分けて管理する。

## ガイドライン

- PDF や画像など大きめのバイナリは Git LFS の導入を検討する
- 回路図を変更する PR では、ビューワで開いたスクリーンショットを PR の説明文に
  添付するとレビューしやすい
- 配線写真は `wiring/` に日付入りファイル名で残すと後から見返しやすい

## 不要な班は

このディレクトリごと削除してよい。
