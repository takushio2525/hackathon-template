# assets — プロジェクト固有データの置き場（例）

プロジェクトで扱う**データアセット**を置く場所の例。

コード以外のリソース（設定ファイル・パラメータ・テストデータなど）をまとめて
管理するためのディレクトリ。用途に合わせてリネームしても問題ない
（例: `data/`, `resources/`, `maps/`, `scenarios/` など）。

## こんなときに使う

- ゲームのマップデータ・シナリオ・パラメータ
- センサのキャリブレーション値・設定ファイル
- 音声・画像・3Dモデルなどのメディアファイル
- テストケース用のサンプルデータ

## 構成

| ファイル / ディレクトリ | 内容 |
|---|---|
| `format_spec.md` | データフォーマットの仕様書（記入テンプレ） |
| `examples/` | 実データのサンプル置き場 |

## 不要な班は

このディレクトリごと削除してよい。
